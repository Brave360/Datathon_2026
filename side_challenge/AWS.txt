AWS SIDE CHALLENGE SUBMISSION
==============================

We deployed the full application on EC2, stored listing images on S3, ran semantic
embedding inference through Amazon Bedrock, and indexed the entire listing corpus 
in Amazon OpenSearch Serverless for real-time KNN vector search. Every search 
query hits AWS infrastructure.

─────────────────────────────────────────────────────────────────────────────────
AWS SERVICES USED
─────────────────────────────────────────────────────────────────────────────────

  [x] Amazon EC2                    — production deployment of the full Docker stack
  [x] Amazon S3                     — listing image storage and URL resolution
  [x] Amazon Bedrock                — model inference (no GPU management)
  [x] Titan Embed Text v2           — 256-dim text embeddings for semantic search
  [x] Titan Embed Image v1          — multimodal embeddings for image search
  [x] OpenSearch Serverless         — KNN vector index for text + image retrieval

─────────────────────────────────────────────────────────────────────────────────
KEY FILES
─────────────────────────────────────────────────────────────────────────────────

  scripts/embed_listings.py           Embeds titles+descriptions via Titan Text v2 
                                      Populate OpenSearch `description` index
  scripts/embed_images.py             Downloads images from S3, embeds via Titan Image v1 
                                      Populate OpenSearch `images` index
  app/participant/soft_filtering.py   Bedrock inference + OpenSearch KNN search
  app/participant/ranking.py          Multi-signal ranker, AWS integration point
  app/core/s3.py                      (Given) S3 image URL resolution via boto3
  docker-compose.yml                  Full stack deployment definition
  Dockerfile                          API container (runs on EC2)
